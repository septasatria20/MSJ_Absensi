Deskripsi Use Case Report – Generate Report Finger

ID Use Case UCR1
Nama Use Case Generate Report Finger
Tujuan Memungkinkan admin/HRD untuk melihat dan menganalisa data raw dari mesin fingerprint dalam format tabel untuk keperluan audit dan verifikasi data absensi.
Aktor Primer Admin, HRD
Aktor Sekunder Sistem Database Absensi
Deskripsi Singkat Use case ini menjelaskan proses generate report data finger mentah yang telah di-pull dari mesin. Report menampilkan semua record absensi dengan detail waktu, lokasi mesin, dan status.
Precondition
  1. Admin/HRD telah login ke sistem
  2. Data fingerprint sudah di-pull ke database
  3. Admin/HRD memiliki akses ke menu Report
Postcondition
  1. Report finger ditampilkan di browser
  2. Admin/HRD dapat export report ke Excel atau PDF
  3. Log akses report tercatat di sistem
Skenario Utama
  1. Admin/HRD mengakses menu "Report > Report Finger"
  2. Sistem menampilkan form filter dengan opsi:
    • Periode tanggal (dari - sampai)
    • NIK atau Nama Karyawan
    • Mesin fingerprint (lokasi)
    • Department/Group
  3. Admin/HRD mengisi filter sesuai kebutuhan
  4. Admin/HRD klik tombol "Generate Report"
  5. Sistem melakukan query ke database
  6. Sistem menampilkan tabel report dengan kolom:
    • No
    • Tanggal & Waktu
    • NIK
    • Nama Karyawan
    • Mesin (Lokasi)
    • Status (Check-in/Check-out)
    • Tipe (Fingerprint/Manual)
  7. Sistem menyediakan fitur sorting dan pagination
  8. Admin/HRD dapat memilih untuk export dengan format Excel atau PDF
  9. Jika export, sistem generate file dan trigger download
  10. Sistem mencatat aktivitas generate report
Skenario Alternatif A1: Data tidak ditemukan
1. Di langkah 6, jika tidak ada data sesuai filter
2. Sistem menampilkan pesan "Data tidak ditemukan untuk periode tersebut"
3. Admin/HRD dapat mengubah filter atau keluar

A2: Data terlalu banyak
1. Di langkah 6, jika data hasil query lebih dari 10.000 record
2. Sistem menampilkan warning untuk mempersempit periode
3. Admin dapat melanjutkan (dengan pagination) atau menyesuaikan filter

A3: Export timeout
1. Di langkah 9, jika proses export memakan waktu lama
2. Sistem menampilkan progress bar
3. Jika timeout, sistem dapat mengirim file via email sebagai alternatif

Deskripsi Use Case Report – Generate Report Kehadiran

ID Use Case UCR2
Nama Use Case Generate Report Kehadiran
Tujuan Memungkinkan admin/HRD untuk melihat rekap kehadiran karyawan dalam periode tertentu yang sudah diproses dan dianalisa untuk keperluan evaluasi dan payroll.
Aktor Primer Admin, HRD
Aktor Sekunder a. Sistem Perhitungan Kehadiran
b. Sistem Database Absensi
Deskripsi Singkat Use case ini menjelaskan proses generate report kehadiran yang menampilkan summary kehadiran per karyawan meliputi jumlah hadir, izin, sakit, alpa, dan statistik lainnya.
Precondition
  1. Admin/HRD telah login ke sistem
  2. Data absensi sudah tersedia dan diproses
  3. Rules perhitungan kehadiran sudah dikonfigurasi
Postcondition
  1. Report kehadiran ditampilkan dengan summary lengkap
  2. Admin/HRD dapat melihat breakdown detail per hari
  3. Report dapat diexport untuk analisa lebih lanjut
Skenario Utama
  1. Admin/HRD mengakses menu "Report > Report Kehadiran"
  2. Sistem menampilkan form filter:
    • Periode (Bulan/Tahun atau custom date range)
    • Department/Divisi
    • Group/Shift
    • NIK atau Nama Karyawan
  3. Admin/HRD memilih periode dan filter lainnya
  4. Admin/HRD klik "Generate Report"
  5. Sistem melakukan perhitungan:
    • Total hari hadir
    • Total hari izin
    • Total hari sakit
    • Total hari alpa
    • Total cuti
    • Persentase kehadiran
  6. Sistem menampilkan report dalam bentuk tabel dengan kolom:
    • No
    • NIK & Nama
    • Department
    • Hadir
    • Izin
    • Sakit
    • Alpa
    • Cuti
    • Persentase (%)
    • Action (Lihat Detail)
  7. Admin/HRD dapat klik "Lihat Detail" untuk melihat breakdown per hari
  8. Sistem menampilkan kalender/tabel detail kehadiran harian karyawan tersebut
  9. Admin/HRD dapat export report ke Excel/PDF
  10. Sistem juga menampilkan grafik summary kehadiran
Skenario Alternatif A1: Report per Department
1. Di langkah 3, jika admin memilih filter per Department tanpa karyawan spesifik
2. Sistem menampilkan summary per department dengan total agregat
3. Admin dapat drill down ke list karyawan per department

A2: View grafik statistik
1. Setelah langkah 6, admin dapat switch ke view grafik
2. Sistem menampilkan pie chart atau bar chart kehadiran
3. Admin dapat switch kembali ke view tabel

A3: Periode belum selesai
1. Di langkah 3, jika periode yang dipilih masih berjalan (bulan ini)
2. Sistem menampilkan info "Data masih berjalan, report dapat berubah"
3. Admin dapat melanjutkan generate report dengan data yang sudah ada

Deskripsi Use Case Report – Generate Report Keterlambatan

ID Use Case UCR3
Nama Use Case Generate Report Keterlambatan
Tujuan Memungkinkan admin/HRD untuk mengidentifikasi dan menganalisa pola keterlambatan karyawan untuk keperluan evaluasi kinerja dan tindakan disiplin.
Aktor Primer Admin, HRD
Aktor Sekunder a. Sistem Perhitungan Keterlambatan
b. Sistem Database Absensi
Deskripsi Singkat Use case ini menjelaskan proses generate report keterlambatan yang menampilkan daftar karyawan yang terlambat, frekuensi, durasi keterlambatan, dan analisa trend untuk membantu pengambilan keputusan tindakan disiplin.
Precondition
  1. Admin/HRD telah login ke sistem
  2. Data absensi dan jadwal shift sudah tersedia
  3. Rules keterlambatan (toleransi waktu) sudah dikonfigurasi
Postcondition
  1. Report keterlambatan ditampilkan dengan detail lengkap
  2. HRD dapat menggunakan data untuk tindakan disiplin
  3. Report dapat diexport atau digunakan untuk generate SP otomatis
Skenario Utama
  1. Admin/HRD mengakses menu "Report > Report Keterlambatan"
  2. Sistem menampilkan form filter:
    • Periode tanggal
    • Department/Divisi
    • Minimum frekuensi keterlambatan
    • Minimum durasi terlambat (menit)
    • NIK atau Nama Karyawan
  3. Admin/HRD mengisi filter yang diinginkan
  4. Admin/HRD klik "Generate Report"
  5. Sistem melakukan analisa keterlambatan:
    • Bandingkan waktu check-in vs jadwal shift
    • Hitung selisih waktu terlambat
    • Hitung frekuensi keterlambatan per karyawan
    • Identifikasi pola (hari tertentu, etc)
  6. Sistem menampilkan report dengan kolom:
    • No
    • NIK & Nama Karyawan
    • Department
    • Tanggal
    • Jadwal Masuk
    • Waktu Check-in Aktual
    • Durasi Terlambat
    • Frekuensi (dalam periode)
    • Status Tindakan
  7. Sistem juga menampilkan summary:
    • Total karyawan yang terlambat
    • Rata-rata durasi keterlambatan
    • Departemen dengan keterlambatan tertinggi
    • Grafik trend keterlambatan
  8. Admin/HRD dapat sorting berdasarkan frekuensi atau durasi
  9. Admin/HRD dapat memilih karyawan dan langsung membuat SP
  10. Admin/HRD dapat export report ke Excel/PDF
  11. Sistem mencatat aktivitas akses report
Skenario Alternatif A1: Filter berdasarkan tingkat severity
1. Di langkah 3, admin dapat memilih filter "Hanya Critical" (>3 kali atau >30 menit)
2. Sistem hanya menampilkan karyawan dengan keterlambatan kritis
3. List berkurang fokus pada yang perlu tindakan segera

A2: Tidak ada keterlambatan
1. Di langkah 6, jika tidak ada data keterlambatan dalam periode tersebut
2. Sistem menampilkan pesan "Tidak ada keterlambatan terdeteksi"
3. Admin dapat mengubah filter atau melihat periode lain

A3: Quick action untuk SP
1. Di langkah 9, jika admin klik "Buat SP"
2. Sistem membuka form Kirim SP dengan data karyawan dan pelanggaran sudah terisi
3. Admin tinggal review dan kirim SP
4. Setelah SP terkirim, status di report berubah menjadi "SP Dikirim"

Deskripsi Use Case Report – Export Report ke Excel/PDF

ID Use Case UCR4
Nama Use Case Export Report ke Excel/PDF
Tujuan Memungkinkan admin/HRD untuk mengexport data report dalam format Excel atau PDF untuk keperluan presentasi, distribusi, atau analisa offline.
Aktor Primer Admin, HRD
Aktor Sekunder a. Sistem Export Service
b. Library Excel (PHPSpreadsheet)
c. Library PDF (mPDF/TCPDF)
Deskripsi Singkat Use case ini menjelaskan proses export report yang sudah di-generate ke format file Excel (.xlsx) atau PDF dengan format yang rapi dan siap digunakan untuk berbagai keperluan.
Precondition
  1. Admin/HRD telah login ke sistem
  2. Report sudah di-generate dan ditampilkan
  3. Admin/HRD memiliki hak akses untuk export
Postcondition
  1. File Excel atau PDF berhasil di-generate
  2. File otomatis ter-download ke komputer user
  3. Log aktivitas export tercatat di sistem
Skenario Utama
  1. Admin/HRD sudah berada di halaman report (Finger/Kehadiran/Keterlambatan)
  2. Report sudah ditampilkan dengan data sesuai filter
  3. Admin/HRD klik tombol "Export"
  4. Sistem menampilkan pilihan format:
    • Excel (.xlsx)
    • PDF (.pdf)
  5. Admin/HRD memilih format yang diinginkan
  6. Jika PDF, sistem menanyakan orientasi (Portrait/Landscape)
  7. Sistem mulai proses generate file:
    • Ambil data dari query atau session
    • Format data sesuai template
    • Tambahkan header (judul report, logo, periode, dll)
    • Susun data dalam tabel yang rapi
    • Tambahkan footer (tanggal export, halaman, dll)
  8. Sistem tampilkan progress bar "Generating file..."
  9. Setelah selesai, sistem trigger download file
  10. File tersimpan di folder download user dengan nama format:
    • ReportKehadiran_2026-02_20260224.xlsx
    • ReportKeterlambatan_2026-02_20260224.pdf
  11. Sistem menampilkan notifikasi "File berhasil didownload"
  12. Sistem mencatat log aktivitas export
Skenario Alternatif A1: Data terlalu besar untuk export
1. Di langkah 7, jika data lebih dari 50.000 record
2. Sistem menampilkan warning "Data terlalu besar, export mungkin memakan waktu lama"
3. Sistem memberikan opsi:
  • Lanjutkan export (all data)
  • Export hanya 1000 record teratas
  • Kirim via email setelah selesai
4. Admin memilih opsi dan proses dilanjutkan

A2: Export gagal/error
1. Di langkah 7, jika terjadi error saat generate file
2. Sistem menampilkan error message dengan detail error
3. Admin dapat retry atau melaporkan ke IT support
4. Sistem mencatat error log untuk troubleshooting

A3: Custom format export
1. Di langkah 4, admin dapat klik "Advanced Options"
2. Sistem menampilkan opsi tambahan:
  • Pilih kolom yang akan diexport
  • Sort order
  • Tambahkan summary/total
  • Sertakan grafik (untuk PDF)
3. Admin kustomisasi sesuai kebutuhan
4. Proses export dilanjutkan dengan setting custom

Catatan Umum untuk Semua Report

Keamanan Data
  • Semua akses report dicatat dengan timestamp dan user ID
  • Admin hanya dapat melihat report sesuai department yang di-assign
  • HRD dapat melihat semua department
  • Data personal (gaji, dll) di-mask jika diakses oleh role tertentu
Performance
  • Report dengan data besar (>5000 record) menggunakan pagination
  • Query database dioptimasi dengan indexing
  • Hasil report dapat di-cache untuk periode yang sudah lewat
  • Export besar dapat dijalankan sebagai background job
Integrasi
  • Report dapat dijadwalkan untuk generate otomatis (scheduled report)
  • Hasil report dapat dikirim ke email secara otomatis
  • Data report dapat diintegrasikan dengan sistem payroll
  • API tersedia untuk integrasi dengan sistem eksternal
