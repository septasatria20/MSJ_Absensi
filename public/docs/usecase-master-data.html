Deskripsi Use Case Master Data - Lihat Daftar Karyawan dan Departemen

ID Use Case UCMD1
Nama Use Case Lihat Daftar Karyawan dan Departemen
Tujuan Memungkinkan HRD untuk melihat dan mengakses daftar karyawan beserta informasi departemen mereka yang diambil dari sistem HRIS perusahaan untuk keperluan monitoring dan manajemen absensi.
Aktor Primer HRD
Aktor Sekunder a. HRIS (Sistem Informasi SDM Perusahaan)
b. Database Sistem Absensi
Deskripsi Singkat Use case ini menjelaskan bagaimana HRD dapat melihat daftar karyawan aktif beserta informasi departemen, jabatan, dan data terkait lainnya yang diambil dari sistem HRIS perusahaan dan ditampilkan dalam format yang mudah dibaca.
Precondition
  1. HRD telah login ke sistem absensi
  2. Koneksi ke sistem HRIS tersedia dan dapat diakses
  3. Data karyawan di HRIS sudah ter-update
Postcondition
  1. Daftar karyawan dan departemen ditampilkan di layar
  2. HRD dapat melihat informasi detail setiap karyawan
  3. Data dapat digunakan untuk proses lain seperti mapping atau filtering
Skenario Utama
  1. HRD login ke sistem
  2. HRD membuka menu "Master Data" atau "Data Karyawan"
  3. Sistem melakukan koneksi ke HRIS perusahaan
  4. Sistem mengambil data karyawan aktif dari HRIS dengan informasi:
    • NIK (Nomor Induk Karyawan)
    • Nama lengkap karyawan
    • Departemen/Divisi
    • Jabatan
    • Status karyawan (Aktif/Non-aktif)
    • Tanggal bergabung
  5. Sistem menampilkan daftar karyawan dalam bentuk tabel
  6. HRD dapat melakukan filter berdasarkan departemen atau jabatan
  7. HRD dapat melakukan pencarian berdasarkan NIK atau nama
  8. HRD dapat melihat detail karyawan dengan klik pada baris data
Skenario Alternatif A1: Koneksi ke HRIS Gagal
  1. Di langkah 3, jika koneksi ke HRIS gagal atau timeout
  2. Sistem menampilkan pesan error "Tidak dapat terhubung ke sistem HRIS"
  3. Sistem menampilkan data karyawan dari cache terakhir (jika ada)
  4. HRD dapat mencoba refresh atau contact IT support

A2: Data Tidak Ditemukan
  1. Di langkah 7, jika pencarian tidak menemukan hasil
  2. Sistem menampilkan pesan "Data tidak ditemukan"
  3. HRD dapat mengubah kata kunci pencarian atau menghapus filter

Deskripsi Use Case Master Data - Lihat Link PIN Karyawan

ID Use Case UCMD2
Nama Use Case Lihat Link PIN Karyawan
Tujuan Memungkinkan HRD untuk melihat dan mengelola mapping PIN fingerprint karyawan dengan data NIK di sistem untuk memastikan data absensi dari mesin fingerprint dapat terintegrasi dengan benar.
Aktor Primer HRD
Aktor Sekunder a. HRIS (Sistem Informasi SDM Perusahaan)
b. FTM (Fingerspot/Mesin Fingerprint)
c. Database Sistem Absensi
Deskripsi Singkat Use case ini menjelaskan bagaimana HRD dapat melihat mapping antara PIN di mesin fingerprint dengan NIK karyawan di sistem HRIS, sehingga data absensi yang ditarik dari FTM dapat diidentifikasi dengan benar ke karyawan yang tepat.
Precondition
  1. HRD telah login ke sistem absensi
  2. Data karyawan dari HRIS sudah tersedia
  3. Data PIN dari mesin fingerprint (FTM) sudah di-pull
Postcondition
  1. Daftar mapping PIN-NIK ditampilkan
  2. HRD dapat mengidentifikasi PIN yang sudah/belum ter-mapping
  3. Data mapping dapat digunakan untuk validasi integrasi data absensi
Skenario Utama
  1. HRD login ke sistem
  2. HRD membuka menu "Master Data > Mapping PIN" atau "Link PIN Karyawan"
  3. Sistem mengambil data karyawan dari HRIS
  4. Sistem mengambil data PIN dari FTM (Fingerspot)
  5. Sistem menampilkan tabel mapping dengan kolom:
    • PIN (ID di mesin fingerprint)
    • NIK (dari HRIS)
    • Nama Karyawan
    • Departemen
    • Status Mapping (Aktif/Belum Mapping)
    • Action (Edit/Update)
  6. HRD dapat melihat PIN mana yang sudah ter-link dengan NIK
  7. HRD dapat filter berdasarkan status mapping atau departemen
  8. HRD dapat melakukan pencarian berdasarkan PIN atau NIK
Skenario Alternatif A1: PIN Belum Ter-mapping
  1. Di langkah 6, jika ditemukan PIN yang belum ter-link dengan NIK
  2. Sistem menampilkan warning atau highlight pada baris tersebut
  3. HRD dapat klik "Edit" untuk melakukan mapping manual
  4. HRD memilih NIK yang sesuai dari dropdown list
  5. Sistem menyimpan mapping baru

A2: Duplikasi PIN atau NIK
  1. Jika sistem mendeteksi duplikasi (satu NIK ter-link ke multiple PIN atau sebaliknya)
  2. Sistem menampilkan alert "Terdeteksi duplikasi mapping"
  3. HRD perlu review dan perbaiki mapping yang error

Deskripsi Use Case Master Data - Lihat Grup Kerja

ID Use Case UCMD3
Nama Use Case Lihat Grup Kerja
Tujuan Memungkinkan HRD untuk melihat dan mengelola pengelompokan karyawan berdasarkan grup kerja atau tim untuk keperluan penugasan jadwal shift dan monitoring kehadiran berdasarkan grup.
Aktor Primer HRD
Aktor Sekunder a. HRIS (Sistem Informasi SDM Perusahaan)
b. Database Sistem Absensi
Deskripsi Singkat Use case ini menjelaskan bagaimana HRD dapat melihat daftar grup kerja yang ada, melihat anggota setiap grup, dan informasi terkait seperti supervisor atau jadwal kerja grup tersebut.
Precondition
  1. HRD telah login ke sistem absensi
  2. Data grup kerja sudah dikonfigurasi di sistem
  3. Data karyawan dari HRIS tersedia
Postcondition
  1. Daftar grup kerja ditampilkan
  2. HRD dapat melihat detail anggota setiap grup
  3. Informasi grup dapat digunakan untuk assignment jadwal atau report
Skenario Utama
  1. HRD login ke sistem
  2. HRD membuka menu "Master Data > Grup Kerja" atau "Group Karyawan"
  3. Sistem mengambil data grup kerja dari database
  4. Sistem menampilkan daftar grup dalam bentuk card atau tabel dengan informasi:
    • Nama Grup Kerja
    • Kode Grup
    • Jumlah Anggota
    • Supervisor/Leader
    • Tipe Kerja (Shift/Non-Shift)
    • Status (Aktif/Non-aktif)
  5. HRD dapat klik pada salah satu grup untuk melihat detail
  6. Sistem menampilkan detail grup dengan daftar anggota karyawan
  7. HRD dapat melakukan filter atau pencarian grup
  8. HRD dapat melihat jadwal shift yang di-assign ke grup tersebut
Skenario Alternatif A1: Grup Tidak Memiliki Anggota
  1. Di langkah 6, jika grup yang dipilih belum memiliki anggota
  2. Sistem menampilkan pesan "Grup ini belum memiliki anggota"
  3. HRD dapat menambahkan anggota atau menutup detail grup

A2: View Berdasarkan Departemen
  1. Di langkah 4, HRD dapat memilih filter "Berdasarkan Departemen"
  2. Sistem mengelompokkan dan menampilkan grup per departemen
  3. HRD dapat expand/collapse setiap departemen untuk melihat grup di dalamnya

Deskripsi Use Case Master Data - Lihat Jadwal Shift

ID Use Case UCMD4
Nama Use Case Lihat Jadwal Shift
Tujuan Memungkinkan HRD untuk melihat dan memonitor jadwal shift karyawan yang sudah dikonfigurasi dalam sistem untuk memastikan pembagian jadwal kerja yang tepat dan sesuai operasional.
Aktor Primer HRD
Aktor Sekunder a. FTM (Fingerspot - untuk validasi waktu absen)
b. Database Sistem Absensi
Deskripsi Singkat Use case ini menjelaskan bagaimana HRD dapat melihat master jadwal shift yang tersedia, detail waktu kerja setiap shift, dan karyawan atau grup mana yang di-assign ke shift tersebut.
Precondition
  1. HRD telah login ke sistem absensi
  2. Data master shift sudah dikonfigurasi di sistem
  3. Data karyawan dan grup kerja tersedia
Postcondition
  1. Daftar jadwal shift ditampilkan dengan detail lengkap
  2. HRD dapat melihat assignment karyawan ke shift tertentu
  3. Informasi shift dapat digunakan untuk validasi absensi dan report
Skenario Utama
  1. HRD login ke sistem
  2. HRD membuka menu "Master Data > Jadwal Shift" atau "Master Shift"
  3. Sistem mengambil data master shift dari database
  4. Sistem menampilkan daftar shift dalam bentuk tabel dengan informasi:
    • Kode Shift (contoh: Shift A, Shift B, Pagi, Siang, Malam)
    • Nama Shift
    • Jam Masuk (Check-in time)
    • Jam Keluar (Check-out time)
    • Total Jam Kerja
    • Toleransi Keterlambatan
    • Jumlah Karyawan Ter-assign
    • Status Shift (Aktif/Non-aktif)
  5. HRD dapat klik pada salah satu shift untuk melihat detail lengkap
  6. Sistem menampilkan detail shift termasuk daftar karyawan/grup yang menggunakan shift tersebut
  7. HRD dapat melihat breakdown waktu (jam masuk, istirahat, pulang)
  8. HRD dapat filter shift berdasarkan status atau tipe (pagi/siang/malam)
Skenario Alternatif A1: View Shift dengan Assignment Karyawan
  1. Di langkah 6, HRD klik "Lihat Karyawan" pada salah satu shift
  2. Sistem menampilkan modal/page baru dengan daftar karyawan yang di-assign ke shift tersebut
  3. HRD dapat melihat NIK, nama, departemen, dan periode assignment
  4. HRD dapat export daftar karyawan per shift jika diperlukan

A2: Tidak Ada Shift Aktif
  1. Di langkah 4, jika tidak ada shift yang berstatus aktif
  2. Sistem menampilkan pesan "Belum ada jadwal shift aktif"
  3. HRD dapat membuat shift baru atau mengaktifkan shift yang non-aktif

Deskripsi Use Case Master Data - Lihat Kalender Shift dan Kalender Non Shift

ID Use Case UCMD5
Nama Use Case Lihat Kalender Shift dan Kalender Non Shift
Tujuan Memungkinkan HRD untuk melihat kalender hari libur dan hari kerja untuk karyawan shift dan non-shift, sehingga dapat mengelola jadwal dengan mempertimbangkan hari libur nasional, cuti bersama, dan pengaturan khusus lainnya.
Aktor Primer HRD
Aktor Sekunder a. FTM (Fingerspot - untuk referensi hari kerja)
b. Database Sistem Absensi
Deskripsi Singkat Use case ini menjelaskan bagaimana HRD dapat melihat kalender kerja yang membedakan antara karyawan shift dan non-shift, termasuk hari libur nasional, cuti bersama, dan hari kerja khusus yang sudah dikonfigurasi dalam sistem.
Precondition
  1. HRD telah login ke sistem absensi
  2. Data kalender dan hari libur sudah dikonfigurasi di sistem
  3. Sistem dapat membedakan karyawan shift dan non-shift
Postcondition
  1. Kalender shift dan non-shift ditampilkan dengan jelas
  2. HRD dapat melihat hari libur, hari kerja, dan status khusus lainnya
  3. Informasi kalender dapat digunakan untuk planning dan scheduling
Skenario Utama
  1. HRD login ke sistem
  2. HRD membuka menu "Master Data > Kalender" atau "Hari Libur"
  3. Sistem menampilkan pilihan view:
    • Kalender Shift (untuk karyawan shift)
    • Kalender Non-Shift (untuk karyawan non-shift)
  4. HRD memilih tipe kalender yang ingin dilihat
  5. Sistem menampilkan kalender dalam format bulanan atau tahunan dengan legend:
    • Hari kerja normal (warna putih/default)
    • Hari libur nasional (warna merah)
    • Cuti bersama (warna kuning)
    • Hari libur khusus shift (warna biru - jika ada)
    • Weekend (warna abu-abu untuk non-shift)
  6. HRD dapat navigasi antar bulan atau tahun
  7. HRD dapat klik pada tanggal tertentu untuk melihat detail (jenis libur, keterangan, dll)
  8. Sistem menampilkan informasi lengkap pada popup atau sidebar
  9. HRD dapat melihat summary:
    • Total hari kerja dalam bulan/tahun
    • Total hari libur
    • Total cuti bersama
Skenario Alternatif A1: Switch Antara Kalender Shift dan Non-Shift
  1. Di langkah 4, HRD dapat dengan mudah switch antara kedua kalender dengan toggle atau tab
  2. Sistem reload kalender sesuai tipe yang dipilih
  3. Perbedaan utama muncul pada weekend (shift tetap kerja, non-shift libur)

A2: Export Kalender
  1. Setelah langkah 5, HRD dapat klik tombol "Export Kalender"
  2. Sistem generate file PDF atau Excel berisi kalender lengkap dengan keterangan
  3. File dapat didownload untuk keperluan sosialisasi atau print

A3: Tidak Ada Data Kalender untuk Tahun Tertentu
  1. Jika HRD navigasi ke tahun yang belum dikonfigurasi
  2. Sistem menampilkan pesan "Data kalender untuk tahun ini belum tersedia"
  3. HRD dapat kembali ke tahun sebelumnya atau request konfigurasi kalender baru

📌 Catatan Umum Use Case Master Data

Semua use case master data bersifat read-only atau view-only dalam konteks ini. HRD dapat melihat dan memonitor data, namun untuk proses Create, Update, Delete (CUD) biasanya dilakukan pada sistem terpisah (HRIS untuk data karyawan, atau konfigurasi sistem untuk shift dan kalender).

Integrasi dengan HRIS dan FTM:

  • Data karyawan dan departemen di-pull dari HRIS secara periodik atau real-time
  • Data PIN dan waktu absensi diambil dari FTM (Fingerspot)
  • Sistem absensi bertindak sebagai middleware yang mengintegrasikan kedua sistem untuk keperluan monitoring dan reporting
