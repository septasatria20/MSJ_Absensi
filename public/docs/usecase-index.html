Tentang Dokumentasi Ini

Dokumentasi use case ini menjelaskan secara detail semua fitur dan proses bisnis dalam sistem manajemen absensi dari perspektif pengguna (Admin dan HRD). Setiap use case dilengkapi dengan diagram, deskripsi lengkap, dan skenario alternatif.

Dokumentasi ini bertujuan untuk memberikan pemahaman yang jelas tentang bagaimana sistem bekerja, siapa yang terlibat, dan apa yang dapat dilakukan di setiap modul.

👥 Aktor dalam Sistem

👨‍💼

Admin

Bertugas mengelola data master, melakukan transaksi absensi harian, dan generate report untuk monitoring kehadiran karyawan.

👩‍💼

HRD

Memiliki akses penuh untuk monitoring, approval, membuat kebijakan, mengirim SP, dan generate report untuk evaluasi karyawan.

📌 Fitur Utama Sistem

🔄 Modul Transaksi

Modul transaksi menangani semua operasi harian terkait absensi, mulai dari pengambilan data dari mesin fingerprint, input manual untuk kasus khusus, approval pertukaran jadwal, identifikasi data missing, hingga pengiriman surat peringatan untuk karyawan yang melanggar.

📈 Modul Report

Modul report menyediakan berbagai laporan analitik untuk membantu manajemen dalam pengambilan keputusan. Laporan meliputi data raw fingerprint, rekap kehadiran bulanan, analisa keterlambatan, dan dapat di-export dalam format Excel atau PDF untuk keperluan presentasi dan arsip.

🎯 Cara Menggunakan Dokumentasi Ini

  1. Pilih kategori use case yang ingin Anda pelajari (Transaksi atau Report)
  2. Baca diagram use case untuk memahami gambaran umum interaksi
  3. Pelajari deskripsi detail setiap use case yang mencakup:
    • Tujuan dan manfaat
    • Aktor yang terlibat
    • Prasyarat dan hasil akhir
    • Langkah-langkah proses (skenario utama)
    • Skenario alternatif untuk kondisi khusus
  4. Gunakan informasi ini sebagai panduan implementasi atau training pengguna
