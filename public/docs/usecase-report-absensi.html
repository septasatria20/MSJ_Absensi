Deskripsi Use Case Report - Lihat Rekap Data FTM

ID Use Case UCR1
Nama Use Case Lihat Rekap Data FTM
Tujuan Memungkinkan HRD untuk melihat laporan rekap data absensi yang berasal langsung dari mesin fingerprint (FTM/Fingerspot) dalam periode tertentu, untuk keperluan monitoring, validasi, dan analisa data mentah dari mesin.
Aktor Primer HRD
Aktor Sekunder a. FTM (Fingerspot/Mesin Fingerprint)
b. Database Sistem Absensi
c. HRIS (untuk data karyawan)
Deskripsi Singkat Use case ini menjelaskan bagaimana HRD dapat mengakses dan melihat rekap data absensi raw yang berasal dari mesin fingerprint. Data ditampilkan dalam format tabel dengan berbagai filter dan dapat digunakan untuk analisa atau troubleshooting masalah data absensi.
Precondition
  1. HRD telah login ke sistem
  2. Data absensi sudah di-pull dari mesin FTM ke database
  3. Data mapping PIN-NIK tersedia
  4. HRD memiliki hak akses untuk melihat report
Postcondition
  1. HRD berhasil melihat rekap data FTM dalam bentuk tabel
  2. Data dapat difilter, disort, dan dianalisa
  3. Log aktivitas view report tercatat
  4. Data siap untuk di-export jika diperlukan
Skenario Utama
  1. HRD login ke sistem
  2. HRD membuka menu "Report > Rekap Data FTM"
  3. Sistem menampilkan halaman report dengan panel filter:
    • Periode (dari tanggal - sampai tanggal)
    • Pilih Mesin FTM (jika multiple mesin)
    • Departemen/Divisi
    • Karyawan (search by NIK/Nama)
    • Status Mapping (All, Mapped, Unmapped)
  4. HRD mengisi filter periode (misalnya: 1-31 Januari 2026)
  5. HRD memilih mesin FTM jika ada lebih dari satu lokasi
  6. HRD dapat mengisi filter lain (departemen, karyawan) atau langsung klik "Tampilkan Data"
  7. Sistem melakukan query ke database untuk mengambil data sesuai filter
  8. Sistem menampilkan tabel rekap data FTM dengan kolom:
    • No (auto-increment)
    • Tanggal
    • Waktu (timestamp dari mesin)
    • PIN (dari mesin fingerprint)
    • NIK (hasil mapping)
    • Nama Karyawan (dari HRIS)
    • Departemen
    • Status Verifikasi (finger/card/password)
    • Mesin (device ID/name)
    • Status Mapping (Mapped/Unmapped)
  9. Di atas tabel, sistem menampilkan summary:
    • Total Records: [jumlah]
    • Total Karyawan Unik: [jumlah]
    • Data Mapped: [jumlah]
    • Data Unmapped: [jumlah]
  10. HRD dapat melakukan sorting dengan klik header kolom (tanggal, PIN, nama, dll)
  11. HRD dapat melakukan pagination (next/previous page) jika data banyak
  12. HRD dapat klik salah satu row untuk melihat detail lengkap record
  13. Sistem menampilkan popup detail dengan informasi:
    • Full timestamp (tahun-bulan-tanggal jam:menit:detik)
    • PIN raw dari mesin
    • Info lengkap karyawan (jika ter-mapping)
    • Device info (mesin ID, lokasi, IP address)
    • Verification method (fingerprint/card/password)
    • Status: apakah data ini sudah diproses atau masih raw
  14. HRD dapat close popup dan kembali ke tabel
  15. HRD dapat mengubah filter untuk melihat data periode lain atau karyawan lain
  16. HRD dapat klik "Refresh Data" untuk reload jika ada data baru yang di-pull
Skenario Alternatif A1: Data Tidak Ditemukan
  1. Di langkah 8, jika tidak ada data yang match dengan filter
  2. Sistem menampilkan message "Tidak ada data ditemukan untuk filter yang dipilih"
  3. Sistem memberikan suggestion:
    • Coba periode tanggal yang lebih luas
    • Cek apakah data sudah di-pull dari mesin
    • Coba filter departemen "All" untuk melihat semua data
  4. HRD dapat adjust filter dan search lagi

A2: Filter Data Unmapped Only
  1. Di langkah 6, HRD dapat pilih filter Status Mapping = "Unmapped"
  2. Sistem hanya tampilkan data dengan PIN yang belum di-mapping ke NIK
  3. HRD dapat melihat daftar PIN yang perlu di-mapping
  4. Dari report ini, HRD dapat langsung navigasi ke menu "Mapping PIN" untuk update

A3: Group by Karyawan
  1. Di panel filter, HRD dapat aktifkan toggle "Group by Karyawan"
  2. Sistem menampilkan data dalam format grouped:
    • Header: NIK - Nama Karyawan
    • Sub-rows: List semua check-in/out karyawan tersebut
    • Sub-total: Jumlah records per karyawan
  3. Format ini memudahkan HRD untuk analisa per individu

A4: Compare Data Antar Mesin
  1. Jika perusahaan punya multiple mesin FTM di lokasi berbeda
  2. HRD dapat pilih "All Machines" di filter
  3. Sistem menampilkan data dari semua mesin dengan kolom tambahan "Lokasi Mesin"
  4. HRD dapat bandingkan data dari mesin utama vs mesin backup
  5. Berguna untuk identifikasi mesin yang bermasalah atau data duplikat

A5: Real-time Monitoring Mode
  1. HRD dapat aktifkan toggle "Auto Refresh" di panel filter
  2. Sistem akan auto refresh data setiap X detik (configurable, default 30 detik)
  3. Berguna untuk monitoring real-time saat jam masuk/pulang karyawan
  4. HRD dapat melihat data terbaru tanpa perlu manual refresh
  5. Ada indicator "Last Updated: [timestamp]" untuk transparansi

Deskripsi Use Case Report - Lihat Rekap Finger

ID Use Case UCR2
Nama Use Case Lihat Rekap Finger
Tujuan Memungkinkan HRD untuk melihat laporan rekap absensi finger yang sudah diproses dan dipasangkan dengan jadwal kerja karyawan, termasuk informasi check-in, check-out, status kehadiran, keterlambatan, dan overtime dalam periode tertentu.
Aktor Primer HRD
Aktor Sekunder a. Database Sistem Absensi
b. HRIS (untuk data karyawan dan jadwal)
c. Sistem Kalender dan Shift
Deskripsi Singkat Use case ini menjelaskan bagaimana HRD dapat melihat rekap absensi yang sudah dipasangkan dengan jadwal kerja dan sudah dihitung status kehadirannya (hadir tepat waktu, terlambat, pulang cepat, overtime, dll). Ini adalah report utama untuk monitoring kehadiran harian karyawan.
Precondition
  1. HRD telah login ke sistem
  2. Data absensi sudah di-pull dan diproses (matched dengan jadwal)
  3. Data jadwal kerja karyawan tersedia
  4. Kalender kerja (hari libur, cuti bersama) sudah dikonfigurasi
  5. HRD memiliki hak akses untuk melihat report
Postcondition
  1. HRD berhasil melihat rekap absensi finger yang sudah diproses
  2. Data ditampilkan dengan perhitungan keterlambatan dan overtime
  3. Summary statistik kehadiran tersedia
  4. Log aktivitas view report tercatat
  5. Data siap untuk analisa lebih lanjut atau export
Skenario Utama
  1. HRD login ke sistem
  2. HRD membuka menu "Report > Rekap Finger" atau "Report > Kehadiran"
  3. Sistem menampilkan halaman report dengan panel filter:
    • Periode (dari tanggal - sampai tanggal) - default: bulan berjalan
    • Departemen/Divisi (dropdown multi-select)
    • Karyawan (search/autocomplete)
    • Status Kehadiran (All, Hadir, Terlambat, Alpa, Izin, Cuti, Sakit)
    • Shift (All, Pagi, Siang, Malam, dst)
    • Tampilan (Harian, Summary per Karyawan)
  4. HRD mengisi filter periode (misalnya: 1-31 Januari 2026)
  5. HRD dapat pilih departemen tertentu atau "All Departments"
  6. HRD dapat pilih tampilan "Harian" (detail per hari) atau "Summary" (ringkasan)
  7. HRD klik "Tampilkan Report"
  8. Sistem query database dan melakukan processing data
  9. Jika pilih tampilan "Harian", sistem menampilkan tabel dengan kolom:
    • Tanggal
    • NIK
    • Nama Karyawan
    • Departemen
    • Shift (Pagi/Siang/Malam/Off)
    • Jadwal Masuk (jam seharusnya)
    • Check-in Aktual
    • Terlambat (menit) - dengan highlight jika > 0
    • Jadwal Pulang
    • Check-out Aktual
    • Pulang Cepat (menit) - jika ada
    • Overtime (jam) - jika ada
    • Total Jam Kerja
    • Status (Hadir/Terlambat/Alpa/Izin/Cuti/Sakit/Libur)
  10. Sistem memberikan color coding untuk status:
    • Hijau: Hadir tepat waktu
    • Kuning: Terlambat < 30 menit
    • Oranye: Terlambat 30-60 menit
    • Merah: Terlambat > 60 menit atau Alpa
    • Biru: Izin/Cuti/Sakit (approved)
    • Abu-abu: Hari Libur/Off
  11. Di atas tabel, sistem menampilkan summary panel:
    • Total Hari Kerja: [jumlah]
    • Total Kehadiran: [jumlah] ([persentase]%)
    • Total Terlambat: [jumlah] orang
    • Total Alpa: [jumlah] orang
    • Rata-rata Keterlambatan: [menit]
    • Total Overtime: [jam]
  12. HRD dapat klik pada salah satu row untuk melihat detail lengkap
  13. Sistem menampilkan popup detail dengan:
    • Info karyawan lengkap (foto, jabatan, contact)
    • Timeline check-in/out dengan foto (jika mesin punya fitur foto)
    • Perhitungan detail keterlambatan/overtime
    • Catatan/keterangan (jika ada input manual)
    • History approval (jika ada izin/adjustment)
  14. HRD dapat sorting by kolom (misal: sort by terlambat terbanyak)
  15. HRD dapat search/filter cepat di tabel
  16. HRD dapat ganti periode atau filter lain untuk melihat data berbeda
Skenario Alternatif A1: Tampilan Summary per Karyawan
  1. Di langkah 6, jika HRD pilih tampilan "Summary per Karyawan"
  2. Sistem menampilkan format summary dengan kolom:
    • NIK
    • Nama Karyawan
    • Departemen
    • Total Hari Kerja
    • Hadir (hari)
    • Terlambat (hari)
    • Alpa (hari)
    • Izin (hari)
    • Cuti (hari)
    • Sakit (hari)
    • Total Keterlambatan (menit)
    • Total Overtime (jam)
    • Persentase Kehadiran (%)
  3. Format ini berguna untuk evaluasi bulanan atau payroll

A2: Filter by Keterlambatan
  1. HRD dapat filter "Hanya Terlambat" untuk fokus pada karyawan yang terlambat
  2. Sistem hanya tampilkan records dengan keterlambatan > 0 menit
  3. HRD dapat sorting dari terlambat terbanyak
  4. Berguna untuk tindak lanjut SP atau counseling

A3: View Data per Departemen (Drill-down)
  1. Sistem dapat menampilkan view "Group by Department"
  2. Setiap departemen ditampilkan dengan summary agregat
  3. HRD dapat expand/collapse untuk lihat detail per karyawan
  4. Tampilan hierarchical: Departemen > Karyawan > Detail harian
  5. Berguna untuk perbandingan performa kehadiran antar departemen

A4: Chart dan Visualisasi
  1. Sistem menyediakan toggle "View Chart"
  2. Sistem generate grafik:
    • Bar chart: Kehadiran vs Terlambat vs Alpa per hari
    • Pie chart: Distribusi status kehadiran
    • Line chart: Trend keterlambatan sepanjang periode
  3. Chart interaktif (hover untuk lihat detail)
  4. HRD dapat export chart sebagai image

A5: Quick Filter Presets
  1. Sistem menyediakan quick filter buttons:
    • "Hari Ini" - auto filter ke tanggal hari ini
    • "Minggu Ini" - auto filter ke 7 hari terakhir
    • "Bulan Ini" - auto filter ke bulan berjalan
    • "Bulan Lalu" - auto filter ke bulan sebelumnya
    • "Problem Cases" - hanya yang alpa/terlambat > 60 menit
  2. HRD cukup klik button tanpa perlu input manual
  3. Meningkatkan efisiensi untuk report rutin

A6: Compare Periode
  1. HRD dapat aktifkan "Compare Mode"
  2. Sistem menampilkan 2 periode side-by-side (misal: Januari vs Februari)
  3. Summary menampilkan delta/perubahan antara periode
  4. Berguna untuk melihat trend improvement atau deterioration

Deskripsi Use Case Report - Lihat Rekap Data Missing

ID Use Case UCR3
Nama Use Case Lihat Rekap Data Missing
Tujuan Memungkinkan HRD untuk melihat laporan lengkap tentang data absensi yang tidak lengkap atau missing (tidak ada check-in, tidak ada check-out, atau tidak ada data sama sekali pada hari kerja) dalam periode tertentu untuk keperluan follow-up dan verifikasi.
Aktor Primer HRD
Aktor Sekunder a. Database Sistem Absensi
b. HRIS (untuk data jadwal kerja)
c. Sistem Kalender
Deskripsi Singkat Use case ini menjelaskan bagaimana HRD dapat melihat rekap lengkap data absensi yang tidak lengkap atau missing, dengan kategorisasi jenis missing (no check-in, no check-out, no data), status verifikasi, dan informasi lain yang diperlukan untuk follow-up kepada karyawan atau supervisor.
Precondition
  1. HRD telah login ke sistem
  2. Data absensi sudah di-pull dan diproses
  3. Data jadwal kerja karyawan tersedia
  4. Sistem sudah melakukan identifikasi data missing
  5. HRD memiliki hak akses untuk melihat report
Postcondition
  1. HRD berhasil melihat rekap data missing dengan lengkap
  2. Data dikategorisasi berdasarkan jenis missing dan status verifikasi
  3. Summary statistik data missing tersedia
  4. HRD dapat langsung navigasi ke verifikasi jika perlu
  5. Log aktivitas view report tercatat
Skenario Utama
  1. HRD login ke sistem
  2. HRD membuka menu "Report > Rekap Data Missing"
  3. Sistem menampilkan halaman report dengan panel filter:
    • Periode (dari tanggal - sampai tanggal)
    • Departemen/Divisi
    • Karyawan (search)
    • Jenis Missing (dropdown): - All - No Check-in Only - No Check-out Only - No Data (Both Missing)
    • Status Verifikasi (dropdown): - All - Belum Verified - Sudah Verified - In Progress
    • Priority (All, High, Medium, Low)
  4. HRD mengisi filter periode (misalnya: 1-15 Januari 2026)
  5. HRD dapat pilih filter jenis missing (misal: hanya "No Check-out")
  6. HRD klik "Tampilkan Report"
  7. Sistem melakukan analisa dan identifikasi data missing berdasarkan:
    • Compare data absensi dengan jadwal kerja
    • Exclude hari libur dan cuti yang sudah approved
    • Kategorisasi jenis missing
    • Check status verifikasi dari history
  8. Sistem menampilkan tabel rekap data missing dengan kolom:
    • Tanggal
    • NIK
    • Nama Karyawan
    • Departemen
    • Jadwal Shift (jam kerja seharusnya)
    • Check-in (ada: tampilkan waktu | kosong: "Missing")
    • Check-out (ada: tampilkan waktu | kosong: "Missing")
    • Jenis Missing (badge: No In/No Out/No Data)
    • Lama Missing (berapa hari sejak kejadian)
    • Status Verifikasi (badge: Pending/Verified/In Progress)
    • Priority (High/Medium/Low - auto atau manual)
    • Action (Quick Verify button)
  9. Sistem memberikan color coding:
    • Merah: Missing > 3 hari dan belum verified (High Priority)
    • Oranye: Missing 1-3 hari dan belum verified (Medium Priority)
    • Kuning: Missing hari ini (Low Priority)
    • Hijau: Sudah verified
  10. Di atas tabel, sistem menampilkan dashboard summary:
    • Total Data Missing: [jumlah] records
    • Karyawan Terdampak: [jumlah] orang
    • No Check-in: [jumlah]
    • No Check-out: [jumlah]
    • No Data: [jumlah]
    • Belum Verified: [jumlah] ([persentase]%)
    • Sudah Verified: [jumlah] ([persentase]%)
  11. Sistem menampilkan mini chart: distribusi missing per departemen (bar chart)
  12. HRD dapat sorting tabel by tanggal, departemen, atau priority
  13. HRD dapat klik salah satu row untuk melihat detail
  14. Sistem menampilkan popup detail dengan:
    • Info lengkap karyawan
    • Jadwal kerja detail untuk tanggal tersebut
    • History absensi karyawan (7 hari terakhir untuk context)
    • Status verifikasi (jika sudah verified, tampilkan by siapa dan alasan)
    • Tombol "Verify Now" untuk langsung verifikasi
  15. HRD dapat close popup dan review record lain
  16. HRD dapat klik "Quick Verify" pada salah satu row untuk verifikasi cepat
  17. Atau HRD dapat select multiple records dan klik "Bulk Verify"
  18. HRD dapat export report untuk share ke supervisor/manager
Skenario Alternatif A1: Filter High Priority Only
  1. HRD dapat filter hanya data missing dengan priority "High"
  2. Sistem tampilkan data yang:
    • Missing > 3 hari DAN belum verified
    • Atau karyawan dengan akumulasi missing > 5x dalam sebulan
  3. Berguna untuk fokus pada yang urgent terlebih dahulu

A2: Group by Karyawan (Problem Employee List)
  1. Sistem dapat toggle view "Group by Employee"
  2. Tampilkan list karyawan dengan jumlah missing terbanyak di periode tersebut
  3. Format:
    • Header: Nama Karyawan - Total Missing: [jumlah]
    • Sub-rows: Detail setiap missing (tanggal, jenis, status)
  4. Berguna untuk identifikasi karyawan yang perlu counseling atau SP

A3: Departemen Comparison View
  1. HRD dapat pilih view "Comparison by Department"
  2. Sistem generate tabel perbandingan:
    • Kolom: Nama Departemen
    • Total Karyawan
    • Total Missing
    • Persentase Missing (%)
    • Rata-rata Missing per Karyawan
    • Trend (naik/turun vs bulan lalu)
  3. Berguna untuk evaluasi performa departemen
  4. Dapat identify departemen yang perlu intervensi

A4: Auto-generate Follow-up List
  1. Sistem menyediakan button "Generate Follow-up List"
  2. Sistem auto-generate dokumen Excel berisi:
    • List karyawan dengan data missing
    • Contact person (nomor HP/email)
    • Nama supervisor
    • Checklist kolom untuk follow-up
  3. HRD dapat print atau share file ini untuk koordinasi

A5: Timeline View
  1. Sistem dapat tampilkan "Timeline View"
  2. Format: Kalender dengan highlighting pada tanggal yang ada missing
  3. Hover tanggal untuk lihat summary missing di hari itu
  4. Click tanggal untuk drill-down ke detail
  5. Visual representation memudahkan identifikasi pattern (misal: banyak missing di hari Senin/Jumat)

A6: Set Reminder for Unverified Data
  1. Untuk data missing yang belum verified > 7 hari
  2. Sistem auto kirim reminder ke HRD via email atau notifikasi dashboard
  3. Reminder berisi summary dan link langsung ke halaman verifikasi
  4. HRD dapat snooze reminder atau mark as in-progress

Deskripsi Use Case Report - Lihat Rekap Surat Peringatan

ID Use Case UCR4
Nama Use Case Lihat Rekap Surat Peringatan
Tujuan Memungkinkan HRD untuk melihat laporan lengkap tentang semua surat peringatan (SP) yang pernah dikirimkan kepada karyawan, termasuk detail SP, status, dan statistics untuk keperluan monitoring, evaluasi, dan audit trail kedisiplinan karyawan.
Aktor Primer HRD
Aktor Sekunder a. Database Sistem Absensi
b. Document Management System
c. HRIS (untuk data karyawan)
Deskripsi Singkat Use case ini menjelaskan bagaimana HRD dapat melihat rekap lengkap history surat peringatan yang telah diterbitkan, dengan filter berdasarkan periode, departemen, tingkat SP, dan status. Report ini penting untuk monitoring kedisiplinan karyawan dan sebagai dokumentasi legal.
Precondition
  1. HRD telah login ke sistem
  2. Data surat peringatan tersimpan di database
  3. Dokumen PDF SP tersimpan di document storage
  4. HRD memiliki hak akses untuk melihat report SP
Postcondition
  1. HRD berhasil melihat rekap surat peringatan
  2. Data ditampilkan dengan lengkap termasuk dokumen attachment
  3. Summary statistics SP tersedia
  4. HRD dapat download dokumen SP yang pernah dikirim
  5. Log aktivitas view report tercatat
Skenario Utama
  1. HRD login ke sistem
  2. HRD membuka menu "Report > Rekap Surat Peringatan"
  3. Sistem menampilkan halaman report dengan panel filter:
    • Periode (dari tanggal - sampai tanggal)
    • Departemen/Divisi
    • Karyawan (search by NIK/Nama)
    • Tingkat SP (All, SP 1, SP 2, SP 3)
    • Jenis Pelanggaran (All, Keterlambatan, Alpa, Izin Tidak Resmi, dll)
    • Status (All, Sent, Draft, Cancelled)
    • Metode Pengiriman (All, Email, Print, Keduanya)
  4. HRD mengisi filter periode (misalnya: Januari - Desember 2025)
  5. HRD dapat pilih filter lain atau langsung klik "Tampilkan Report"
  6. Sistem query database dan retrieve data SP sesuai filter
  7. Sistem menampilkan tabel rekap SP dengan kolom:
    • No Surat
    • Tanggal Surat
    • Tanggal Dikirim
    • NIK
    • Nama Karyawan
    • Departemen
    • Jabatan
    • Tingkat SP (badge: SP 1/SP 2/SP 3)
    • Jenis Pelanggaran
    • Periode Pelanggaran (dari tanggal - sampai tanggal)
    • Dikirim oleh (nama HRD yang mengirim)
    • Metode (Email/Print/Both)
    • Status (badge: Sent/Draft/Cancelled)
    • Action (View, Download PDF, Resend)
  8. Sistem memberikan color coding untuk tingkat SP:
    • Kuning: SP 1 (peringatan pertama)
    • Oranye: SP 2 (peringatan kedua)
    • Merah: SP 3 (peringatan terakhir sebelum PHK)
  9. Di atas tabel, sistem menampilkan dashboard summary:
    • Total SP Dikirim: [jumlah] surat
    • SP 1: [jumlah] ([persentase]%)
    • SP 2: [jumlah] ([persentase]%)
    • SP 3: [jumlah] ([persentase]%)
    • Karyawan Terdampak: [jumlah] orang unik
    • Jenis Pelanggaran Terbanyak: [nama pelanggaran]
    • Departemen Terbanyak SP: [nama departemen]
  10. Sistem menampilkan mini chart:
    • Bar chart: Jumlah SP per bulan (trend)
    • Pie chart: Distribusi jenis pelanggaran
    • Bar chart: Distribusi SP per departemen
  11. HRD dapat sorting tabel by tanggal, tingkat SP, atau departemen
  12. HRD dapat klik salah satu row untuk melihat detail SP
  13. Sistem menampilkan popup detail dengan:
    • Header: Nomor dan Tanggal Surat
    • Info Karyawan (foto, NIK, nama, jabatan, departemen)
    • Tingkat SP dan Jenis Pelanggaran
    • Detail Pelanggaran (daftar tanggal, jenis, durasi)
    • Preview dokumen PDF SP
    • History Pengiriman: - Tanggal kirim - Metode (email/print) - Status email (delivered/bounced/opened) - Dikirim ke: [email karyawan] - CC: [email atasan]
    • History SP Sebelumnya (jika ada)
    • Catatan HRD
    • Created by & Modified by (audit trail)
  14. HRD dapat download PDF dari popup detail
  15. HRD dapat klik "Resend" jika perlu kirim ulang email
  16. HRD dapat close popup dan review SP lain
  17. HRD dapat export seluruh report ke Excel untuk analisa lebih lanjut
Skenario Alternatif A1: Filter by Karyawan (Individual SP History)
  1. HRD dapat search karyawan tertentu di filter
  2. Sistem tampilkan semua history SP karyawan tersebut
  3. Tampilan chronological dari SP pertama sampai terakhir
  4. Berguna untuk review individual sebelum memberikan SP berikutnya
  5. Atau untuk persiapan dokumen jika akan ada PHK

A2: View Top Offenders

  2. Tampilan agregat per departemen:
    • Nama Departemen
    • Total Karyawan
    • Total SP
    • SP per Karyawan (rata-rata)
    • Persentase SP vs total karyawan
    • Trend (naik/turun vs periode sebelumnya)
  3. HRD dapat identify departemen dengan masalah kedisiplinan
  4. Data dapat digunakan untuk evaluasi supervisor/manager

A4: Violation Type Analysis

  2. X-axis: Bulan, Y-axis: Jumlah SP
  3. Dapat compare multiple tahun (overlay)
  4. Identify seasonal pattern (misal: SP meningkat di bulan tertentu)

A7: Bulk Download SP Documents

Deskripsi Use Case Report - Atur Layout Filter

ID Use Case UCR5
Nama Use Case Atur Layout Filter
Tujuan Memungkinkan HRD untuk mengkustomisasi tampilan dan layout filter pada halaman report sesuai preferensi dan kebutuhan, serta menyimpan preset filter yang sering digunakan untuk meningkatkan efisiensi kerja.
Aktor Primer HRD
Aktor Sekunder a. Database Sistem Absensi (untuk simpan preferensi)
b. User Preference Service
Deskripsi Singkat Use case ini menjelaskan bagaimana HRD dapat mengatur layout panel filter pada setiap report (posisi, visibility, default values), membuat dan menyimpan filter preset untuk quick access, dan mengelola preferensi tampilan report agar sesuai dengan workflow mereka.
Precondition
  1. HRD telah login ke sistem
  2. HRD sedang berada di salah satu halaman report
  3. Sistem menyediakan fitur customization filter
  4. HRD memiliki hak akses untuk save preferensi
Postcondition
  1. Layout filter tersimpan sesuai preferensi HRD
  2. Filter preset tersimpan dan dapat digunakan kembali
  3. Preferensi tampilan auto-load setiap kali HRD akses report
  4. Setting dapat di-reset ke default jika diperlukan
Skenario Utama
  1. HRD sudah berada di salah satu halaman report (misal: Report Rekap Finger)
  2. HRD klik icon "⚙️ Atur Filter" atau "Customize Layout" di panel filter
  3. Sistem menampilkan popup "Filter Layout Settings" dengan tabs:
    • Tab "Layout" - untuk atur posisi dan visibility
    • Tab "Presets" - untuk manage saved filters
    • Tab "Defaults" - untuk atur default values
  4. Scenario A: Atur Layout
  5. HRD di tab "Layout"
  6. Sistem menampilkan list semua filter fields yang tersedia:
    • Checkbox untuk show/hide field
    • Drag handle untuk reorder
    • Preview panel di sebelah untuk lihat hasil
  7. HRD dapat:
    • Check/uncheck untuk show/hide filter tertentu
    • Drag & drop untuk ubah urutan filter
    • Pilih layout orientation (Horizontal/Vertical/Grid)
    • Atur collapse/expand behavior (auto-collapse setelah filter)
  8. HRD melihat preview perubahan real-time
  9. HRD klik "Apply Layout"
  10. Sistem save preferensi layout ke user profile
  11. Sistem apply perubahan langsung ke panel filter saat ini
  12. Scenario B: Simpan Filter Preset
  13. HRD sudah mengisi beberapa filter di panel filter (misal: Periode = Bulan ini, Departemen = IT, Status = Hadir)
  14. HRD klik button "💾 Simpan Preset" atau "Save Filter"
  15. Sistem menampilkan dialog "Save Filter Preset"
  16. Form berisi:
    • Nama Preset (required): [input text]
    • Deskripsi (optional): [textarea]
    • Make Default (checkbox): Apply preset ini setiap buka page
    • Share with Team (checkbox): Preset dapat dipakai user HRD lain
  17. HRD mengisi nama preset (misal: "Rekap IT Minggu Ini")
  18. HRD dapat centang "Make Default" jika ingin preset ini auto-load
  19. HRD klik "Save Preset"
  20. Sistem save preset dengan semua filter values saat ini
  21. Sistem menampilkan notifikasi "Preset berhasil disimpan"
  22. Preset muncul di dropdown "Quick Filters" untuk next usage
  23. Scenario C: Load Filter Preset
  24. Next time HRD buka report page
  25. HRD klik dropdown "Quick Filters" atau "📂 Load Preset"
  26. Sistem menampilkan list saved presets:
    • My Presets (yang dibuat user sendiri)
    • Shared Presets (dari team)
    • System Presets (default system: Hari Ini, Minggu Ini, Bulan Ini, dll)
  27. HRD memilih preset "Rekap IT Minggu Ini"
  28. Sistem auto-fill semua filter sesuai preset
  29. HRD tinggal klik "Tampilkan Report" tanpa perlu input manual
  30. Scenario D: Manage Presets
  31. HRD klik "Kelola Preset" atau icon edit di dropdown presets
  32. Sistem menampilkan popup "Manage Filter Presets"
  33. Tampilkan tabel list presets dengan kolom:
    • Nama Preset
    • Deskripsi
    • Created Date
    • Last Used
    • Default (badge jika preset adalah default)
    • Shared (badge jika shared ke team)
    • Action (Edit, Delete, Duplicate)
  34. HRD dapat:
    • Edit preset (ubah nama, values, atau setting)
    • Delete preset yang tidak dipakai lagi
    • Duplicate preset untuk buat variasi baru
    • Set/unset sebagai default
  35. HRD save perubahan dan close popup
Skenario Alternatif A1: Reset Layout to Default
  1. Jika HRD tidak suka dengan custom layout yang sudah dibuat
  2. HRD dapat klik "Reset to Default" di tab Layout
  3. Sistem menampilkan konfirmasi
  4. Sistem restore layout ke setting default sistem
  5. Custom presets tetap tersimpan (tidak hilang)

A2: Import/Export Presets
  1. Untuk sharing presets antar user atau backup
  2. HRD dapat klik "Export Presets" di manage popup
  3. Sistem generate JSON file berisi semua presets
  4. User lain dapat klik "Import Presets" dan upload file tersebut
  5. Sistem load dan save presets ke profile user tersebut

A3: Preset Conflict (Nama Duplikat)
  1. Di langkah save preset, jika nama yang diinput sudah ada
  2. Sistem menampilkan warning "Preset dengan nama ini sudah ada"
  3. HRD dapat pilih:
    • "Overwrite" - replace preset lama
    • "Rename" - ganti nama preset baru
    • "Cancel" - batal save

A4: Shared Preset Permissions
  1. Jika HRD centang "Share with Team"
  2. Sistem cek role HRD (hanya role tertentu boleh share)
  3. Jika allowed, preset visible untuk semua user di team/departemen
  4. User lain dapat load preset tapi tidak bisa edit/delete (read-only)
  5. Hanya creator yang bisa edit/delete shared preset

A5: Auto-load Default Preset
  1. Jika HRD punya preset yang di-set sebagai default
  2. Setiap kali HRD buka halaman report tersebut
  3. Sistem auto-load dan apply preset default
  4. HRD langsung lihat data tanpa perlu set filter manual
  5. HRD masih bisa modify filter jika perlu data berbeda

A6: Column Visibility & Order Customization
  1. Selain filter, HRD juga dapat customize kolom tabel report
  2. Klik icon "Atur Kolom" atau "⚙️" di header tabel
  3. Popup menampilkan list semua kolom dengan checkbox
  4. HRD dapat:
    • Hide/show kolom tertentu
    • Reorder kolom dengan drag & drop
    • Set lebar kolom (auto/fixed)
  5. Setting tersimpan per user dan auto-apply

Deskripsi Use Case Report - Ekspor Data Excel/PDF/Print

ID Use Case UCR6
Nama Use Case Ekspor Data Excel/PDF/Print
Tujuan Memungkinkan HRD untuk mengekspor data report absensi ke berbagai format (Excel, PDF, CSV) atau mencetak langsung untuk keperluan dokumentasi, sharing, analisa offline, atau presentasi ke management.
Aktor Primer HRD
Aktor Sekunder a. Database Sistem Absensi
b. Export Service (Excel/PDF Generator)
c. Print Service
Deskripsi Singkat Use case ini menjelaskan bagaimana HRD dapat mengekspor data report yang sedang ditampilkan ke format Excel, PDF, atau CSV, dengan opsi customization (pilih kolom, apply styling, add header/footer), serta melakukan print langsung ke printer untuk keperluan hard copy.

  1. HRD telah login ke sistem
  2. HRD sedang berada di halaman report dengan data yang sudah ditampilkan
  3. Export service tersedia dan berfungsi
  4. HRD memiliki hak akses untuk export data

    • 🟢 Export to Excel (.xlsx)
    • 🔴 Export to PDF (.pdf)
    • 📋 Export to CSV (.csv)
    • 🖨️ Print
    • ⚙️ Export with Options (advanced)
  4. Scenario A: Quick Export to Excel

  6. Sistem menampilkan loading indicator "Generating Excel file..."
  7. Sistem melakukan:
    • Ambil semua data yang sedang ditampilkan (sesuai filter)
    • Generate file Excel dengan format: - Sheet 1: Data tabel utama - Sheet 2: Summary/Statistics (jika ada) - Header dengan logo perusahaan dan judul report - Auto-width columns - Freeze top row (header kolom) - Basic formatting (border, alternating row colors)
  8. Sistem generate filename: [NamaReport]_[Periode]_[Timestamp].xlsx
    Contoh: RekapFinger_Jan2026_20260125_143055.xlsx
  9. Sistem trigger auto-download file ke browser
  10. Sistem menampilkan notifikasi "File Excel berhasil di-download"
  11. HRD dapat buka file Excel untuk analisa offline
  12. Scenario B: Export to PDF
  13. HRD klik "Export to PDF"
  14. Sistem menampilkan loading "Generating PDF file..."
  15. Sistem generate PDF dengan format:

    • Auto page-break jika data banyak
    • Footer setiap page: Page number, tanggal generate, generated by [nama HRD]
    • Landscape orientation (lebih luas untuk tabel)
  16. Sistem generate filename: [NamaReport]_[Periode]_[Timestamp].pdf
  17. Sistem trigger download file PDF
  18. Sistem menampilkan notifikasi "File PDF berhasil di-download"
  19. HRD dapat buka PDF untuk share atau print
  20. Scenario C: Export with Advanced Options
  21. HRD klik "Export with Options"
  22. Sistem menampilkan popup "Export Settings" dengan form:
    • Format (radio button): Excel / PDF / CSV
    • Pilih Kolom (checkboxes): List semua kolom, HRD bisa pilih mana yang di-export
    • Include Summary (checkbox): Tambahkan sheet/page summary

Skenario Alternatif A1: Export Data Terlalu Besar

  5. Sistem proses dan notifikasi saat selesai

  3. Sistem log error untuk troubleshooting

    • "Retry" - coba export lagi

  1. Jika tabel menggunakan pagination dan hanya sebagian data yang visible
  2. Di export menu, ada option:
    • "Export Current Page Only" - hanya data yang terlihat di page saat ini

  3. HRD pilih sesuai kebutuhan
  4. Sistem export sesuai pilihan

  2. Sistem generate Excel dengan 2 sheets:
    • Sheet 1: Raw data
    • Sheet 2: Pre-configured pivot table untuk analisa
  3. HRD dapat langsung analisa data dengan pivot tanpa perlu setup manual

A6: Branding dan Watermark
  1. For export PDF atau Excel yang akan di-share keluar
  2. HRD dapat add company branding:
    • Logo perusahaan di header
    • Color scheme sesuai corporate identity
    • Watermark "CONFIDENTIAL" atau "INTERNAL USE ONLY"
    • Footer dengan disclaimer atau copyright notice
  3. Sistem apply branding template yang sudah dikonfigurasi
  4. Output lebih professional dan official

📊 Catatan Umum Use Case Report

Karakteristik Report Module:
Use case report bersifat read-only/view-only dan fokus pada visualisasi data untuk analisa, monitoring, dan decision making. Tidak mengubah data di database, hanya menampilkan dalam berbagai format dan perspektif.

Best Practices untuk Report:

  • Performance Optimization: Implementasi caching untuk report yang sering diakses, pagination untuk data besar
  • Filter Flexibility: Sediakan berbagai kombinasi filter agar user dapat slice-and-dice data sesuai kebutuhan
  • Visual Clarity: Gunakan color coding, charts, dan summary stats untuk memudahkan interpretasi data
  • Export Options: Multiple format export (Excel, PDF, CSV) untuk berbagai use case
  • User Preference: Simpan preferensi filter dan layout untuk meningkatkan efisiensi

Integrasi dengan Module Lain:
  • Report "Data Missing" connect ke Transaksi "Verifikasi" untuk quick action
  • Report "Surat Peringatan" connect ke Transaksi "Kirim SP" untuk follow-up
  • Semua report bisa di-export untuk offline analysis atau presentation

Tips untuk HRD:
  • Gunakan filter preset untuk report rutin agar tidak perlu setup berulang
  • Manfaatkan schedule export untuk automatic report bulanan ke management
  • Bandingkan data antar periode untuk track improvement atau identify issues
  • Export dan archive report penting untuk audit trail dan compliance
