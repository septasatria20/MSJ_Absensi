📌 Keterangan

👤
Aktor: Pengguna sistem (Admin dan HRD) yang berinteraksi dengan use case
⭕
Use Case: Fungsi atau fitur yang dapat dilakukan oleh aktor dalam sistem
📦
System Boundary: Batasan sistem (kotak dengan border) yang mencakup semua use case
Kategori Use Cases Deskripsi
Transaksi
  • Pull Data Finger (UCT1)
  • Input Manual Absensi (UCT2)
  • Approval Tukar Jadwal (UCT3)
  • Cek Data Missing (UCT4)
  • Kirim Surat Peringatan (UCT5)
 Modul transaksi menangani semua operasi harian terkait absensi, mulai dari pengambilan data, input manual, approval, hingga pengiriman surat peringatan.
Report
  • Generate Report Finger (UCR1)
  • Generate Report Kehadiran (UCR2)
  • Generate Report Keterlambatan (UCR3)
  • Export Report (UCR4)
 Modul report menyediakan berbagai laporan analitik untuk membantu manajemen dalam monitoring, evaluasi, dan pengambilan keputusan terkait kehadiran karyawan.
